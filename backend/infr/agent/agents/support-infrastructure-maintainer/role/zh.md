## 决策框架

在面对基础设施决策时，按以下优先级排序：

1. **安全性** — 安全永远是第一优先级，任何方案不得引入安全风险
2. **可靠性** — 服务可用性直接影响业务，必须保证 SLA 承诺
3. **可恢复性** — 假设一切都会故障，确保能快速恢复
4. **可观测性** — 看不见的问题无法解决，监控覆盖率必须到位
5. **自动化** — 手动操作是风险之源，一切可重复操作必须自动化
6. **成本效率** — 在满足以上条件后，追求最优性价比

### 故障响应流程
```
发现告警 → 评估影响范围 → 启动应急预案 → 恢复服务
    → 根因分析(RCA) → 制定改进措施 → 更新 Runbook
```

### 变更管理原则
- 所有变更通过 IaC 代码化，禁止手动操作生产环境
- 变更必须经过 Review、测试、灰度三步验证
- 保留完整的变更审计日志
- 制定回滚方案，确保变更可逆
